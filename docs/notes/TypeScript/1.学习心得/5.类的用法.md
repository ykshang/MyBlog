## 一、简介

### 1、声明方式

我们一般使用`class`关键词来声明一个类。

```ts
// 显式类
class Point {
  x: number;
  y: number;
  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }
}
```

也可以不指定类型，让 TypeScript 自动推断类型。

```ts
class Point {
  x;
  y;
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }
}
```

类也可以指定初始值。我们可以直接在属性后面指定初始值，也可以像 Javascript 一样，在构造函数的参数列表中指定初始值。

```ts
class Point {
  x = 0;
  y = 0;
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }
}
// 或者
class Point {
  x: number;
  y: number;
  constructor(x: number = 0, y: number = 0) {
    this.x = x;
    this.y = y;
  }
}
```

::: tip TypeScript 有一个配置项 `strictPropertyInitialization`，只要打开（默认是打开的），就会检查属性是否设置了初值，如果没有就报错。

:::

### 2、类的成员

类的成员主要以下三种：属性、方法、构造器。此外，还有一些特性，其实都是基于这三种成员提供的扩展功能。

我们在定义类的成员时，需要给出具体实现。

```ts
class Point {
  x: number;
  y: number;
  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }
  sepak() {
    console.log("Hello, World!");
  }
}
```

其实方法也是一种特殊的属性，只不过它的类型是函数类型。

如果使用以下写法，需要在实例化的时候，给出具体的实现。

```ts
class Demo {
  sepak: () => void;
  constructor(sepak: () => void) {
    this.sepak = sepak;
  }
}
let fun_param = () => {
  console.log("Hello, World!");
};
let point = new Demo(fun_param);
```

## 二、高级特性

### 1、存取器（`getter`/`setter`）

存取器（`accessor`）是特殊的类方法，可以在读取或写入属性时执行额外的逻辑

- 取值器（`getter`）: 用于读取属性时，可以对属性进行额外的处理，包括转换、验证等。
- 存值器（`setter`）: 用于写入属性时，执行额外的逻辑，包括写入值的验证、修改。

例如以下代码中，我们定义了一个`prop`属性的`getter`和`setter`，方法。

```ts title="demo.ts"
class C {
  _prop = "init value";
  get prop() {
    console.log("读取 prop 属性：", this._prop);
    return this._prop;
  }
  set prop(value) {
    console.log("设置 prop 属性：", value);
    this._prop = value;
  }
}
// 实例化
let c = new C();
c.prop = "123"; // 调用 setter
console.log("打印 prop：", c.prop); // 调用 getter
```

输出：

```console
设置 prop 属性： 123              demo.ts:8
读取 prop 属性： 123              demo.ts:4
打印 prop： 123                  demo.ts:15
```

::: tip 存取器的一些规则

1. 如果某个属性只有`get`，没有`set`，则该属性是只读的。
2. TypeScript 5.1 版之前，`set`方法的参数类型，必须兼容`get`方法的返回值类型，否则报错。5.1 版之后，二者可以不兼容。
3. `get`和`set`方法的可访问性必须相同，要么都是公开的，要么都是私有的。

:::

### 2、属性索引

类允许定义属性索引，主要的使用场景就是：用做类型来定义对象的结构。

下面示例中，`[prop:string]`表示所有属性名类型为字符串的属性，它们的属性值要么是布尔值，要么是返回布尔值的函数。

```ts
class MyClass {
  [prop: string]: boolean | ((prop: string) => boolean);

  getResultById(value: string) {
    return value === "abc";
  }
}
let a: MyClass = {
  prop1: true,
  prop2: (val: string) => val.length > 0,
  getResultById(val: string) {
    return val === "abc";
  },
};
```

### 4、可访问性修饰符

类的内部成员的外部可访问性，由三个可访问性修饰符（access modifiers）控制：`public`、`private`和`protected`。

1. `public`：公开成员，类的实例可以自由访问这个成员。默认可以省略不写。
2. `protected`：受保护成员，只能用在当前类的内部，以及子类的内部，类的实例不能使用该成员。
3. `private`：私有成员，只能用在当前类的内部，类的实例和子类都不能使用该成员。

:::warning 注意

- 子类不能定义父类私有成员的同名成员。
- 正常情况下，除非为了醒目和代码可读性，`public`都是省略不写的。
- `public`修饰符是默认修饰符，如果省略不写，就相当于默认该成员是`public`。
- 这三个修饰符的位置，写在属性或方法的最前面。
  
:::


上面示例中，在类`A`内部，`A`的实例对象可以获取私有成员`x`。

严格地说，`private`定义的私有成员，并不是真正意义的私有成员。一方面，编译成 JavaScript 后，`private`关键字就被剥离了，这时外部访问该成员就不会报错。另一方面，由于前一个原因，TypeScript 对于访问`private`成员没有严格禁止，使用方括号写法（`[]`）或者`in`运算符，实例对象就能访问该成员。

```ts
class A {
  private x = 1;
}

const a = new A();
a["x"]; // 1

if ("x" in a) {
  // 正确
  // ...
}
```

上面示例中，`A`类的属性`x`是私有属性，但是实例使用方括号，就可以读取这个属性，或者使用`in`运算符检查这个属性是否存在，都可以正确执行。

由于`private`存在这些问题，加上它是 ES2022 标准发布前出台的，而 ES2022 引入了自己的私有成员写法`#propName`。因此建议不使用`private`，改用 ES2022 的写法，获得真正意义的私有成员。

```ts
class A {
  #x = 1;
}

const a = new A();
a["x"]; // 报错
```

上面示例中，采用了 ES2022 的私有成员写法（属性名前加`#`），TypeScript 就正确识别了实例对象没有属性`x`，从而报错。

构造方法也可以是私有的，这就直接防止了使用`new`命令生成实例对象，只能在类的内部创建实例对象。

这时一般会有一个静态方法，充当工厂函数，强制所有实例都通过该方法生成。

```ts
class Singleton {
  private static instance?: Singleton;

  private constructor() {}

  static getInstance() {
    if (!Singleton.instance) {
      Singleton.instance = new Singleton();
    }
    return Singleton.instance;
  }
}

const s = Singleton.getInstance();
```

上面示例使用私有构造方法，实现了单例模式。想要获得 Singleton 的实例，不能使用`new`命令，只能使用`getInstance()`方法。

### 3、protected

`protected`修饰符表示该成员是保护成员，只能在类的内部使用该成员，实例无法使用该成员，但是子类内部可以使用。

```ts
class A {
  protected x = 1;
}

class B extends A {
  getX() {
    return this.x;
  }
}

const a = new A();
const b = new B();

a.x; // 报错
b.getX(); // 1
```

上面示例中，类`A`的属性`x`是保护成员，直接从实例读取该属性（`a.x`）会报错，但是子类`B`内部可以读取该属性。

子类不仅可以拿到父类的保护成员，还可以定义同名成员。

```ts
class A {
  protected x = 1;
}

class B extends A {
  x = 2;
}
```

上面示例中，子类`B`定义了父类`A`的同名成员`x`，并且父类的`x`是保护成员，子类将其改成了公开成员。`B`类的`x`属性前面没有修饰符，等同于修饰符是`public`，外界可以读取这个属性。

在类的外部，实例对象不能读取保护成员，但是在类的内部可以。

```ts
class A {
  protected x = 1;

  f(obj: A) {
    console.log(obj.x);
  }
}

const a = new A();

a.x; // 报错
a.f(a); // 1
```

上面示例中，属性`x`是类`A`的保护成员，在类的外部，实例对象`a`拿不到这个属性。但是，实例对象`a`传入类`A`的内部，就可以从`a`拿到`x`。

### 5、静态成员

类的内部可以使用`static`关键字，定义静态成员。

静态成员是只能通过类本身使用的成员，不能通过实例对象使用。

静态成员类似于：`Array.from()`、`Array.isArray()`、`Math.PI`等内置方法

```ts twoslash
// @errors: 2576
class MyClass {
  static x = 0;
  static printX() {
    console.log(MyClass.x);
  }
}

MyClass.x; // 0
MyClass.printX(); // 0
// 实例化对象
let myClass = new MyClass();
myClass.x;
myClass.printX();
```

`static`关键字前面可以使用 `public`、`private`、`protected` 修饰符。

```ts
class MyClass {
  private static x = 0;
}

MyClass.x; // 报错
```

上面示例中，静态属性`x`前面有`private`修饰符，表示只能在`MyClass`内部使用，如果在外部调用这个属性就会报错。

静态私有属性也可以用 ES6 语法的`#`前缀表示，上面示例可以改写如下。

```ts
class MyClass {
  static #x = 0;
}
```

`public`和`protected`的静态成员可以被继承。

```ts
class A {
  public static x = 1;
  protected static y = 1;
}

class B extends A {
  static getY() {
    return B.y;
  }
}

B.x; // 1
B.getY(); // 1
```

上面示例中，类`A`的静态属性`x`和`y`都被`B`继承，公开成员`x`可以在`B`的外部获取，保护成员`y`只能在`B`的内部获取。

### 8、匿名类（类表达式）

匿名类是指没有名称的类。一般使用类表达式来定义匿名类。

```ts
// 匿名类
let Point = class {
```

**二者区别：**

| **特性**   | **显式类**                                 | **匿名类**                 |
| ---------- | ------------------------------------------ | -------------------------- |
| 类名       | 有正式类名（如 Person）                    | 无正式类名，只有变量名     |
| 类型系统   | 类名即类型（Person 类型）                  | 结构类型（通过变量名引用） |
| 可访问性   | 可在定义前引用（hoisting）                 | 必须先定义后使用           |
| 调试信息   | 显示实际类名                               | 显示为 class 或变量名      |
| 继承       | 可直接继承（class Student extends Person） | 只能通过变量名继承         |
| 导出/导入  | 可直接导出类                               | 只能导出变量               |
| 构造函数名 | 与类名相同                                 | 显示为 "class"             |
