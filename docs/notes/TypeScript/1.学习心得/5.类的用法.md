## 一、简介

### 1、声明方式

我们一般使用`class`关键词来声明一个类。

```ts
// 显式类
class Point {
  x: number;
  y: number;
  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }
}
```

也可以不指定类型，让 TypeScript 自动推断类型。

```ts
class Point {
  x;
  y;
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }
}
```

类也可以指定初始值。我们可以直接在属性后面指定初始值，也可以像 Javascript 一样，在构造函数的参数列表中指定初始值。

```ts
class Point {
  x = 0;
  y = 0;
  constructor(x, y) {
    this.x = x;
    this.y = y;
  }
}
// 或者
class Point {
  x: number;
  y: number;
  constructor(x: number = 0, y: number = 0) {
    this.x = x;
    this.y = y;
  }
}
```

::: tip TypeScript 有一个配置项 `strictPropertyInitialization`，只要打开（默认是打开的），就会检查属性是否设置了初值，如果没有就报错。

:::

### 2、类的成员

类的成员主要以下三种：属性、方法、构造器。此外，还有一些特性，其实都是基于这三种成员提供的扩展功能。

我们在定义类的成员时，需要给出具体实现。

```ts
class Point {
  x: number;
  y: number;
  constructor(x: number, y: number) {
    this.x = x;
    this.y = y;
  }
  sepak() {
    console.log("Hello, World!");
  }
}
```

其实方法也是一种特殊的属性，只不过它的类型是函数类型。

如果使用以下写法，需要在实例化的时候，给出具体的实现。

```ts
class Demo {
  sepak: () => void;
  constructor(sepak: () => void) {
    this.sepak = sepak;
  }
}
let fun_param = () => {
  console.log("Hello, World!");
};
let point = new Demo(fun_param);
```

### 3、存取器（`getter`/`setter`）

类的成员还可以包含存取器（`accessor`），他是一直是特殊的方法，主要用作读取或写入属性时执行额外的逻辑，比如验证或转换。

#### 1、概念

- 取值器（`getter`）: 用于读取属性时，可以对属性进行额外的处理，包括转换、验证等。
- 存值器（`setter`）: 用于写入属性时，执行额外的逻辑，包括写入值的验证、修改。

#### 2、规则

以下是一些关于存取器的规则：

::: note 规则 1：如果某个属性对应的只有`get`，没有`set`，则该属性是只读的。
:::

::: note 规则 2：`get`和`set`方法的可访问性修饰符必须相同，要么都是公开的，要么都是私有的。
:::

::: note 规则 3：note 在 `5.1` 版本之前，TypeScript 要求`set`的类型必须兼容`get`的的类型，否则报错。`5.1` 版本之后，这个限制被放宽，允许二者类型不兼容。
什么是类型兼容： 就是`A = B`时，`B`的类型必须是`A`的子类型。`B`中包含`A`的所有属性和方法。
:::

规则 3 主要涉及到一种特殊的情况：DOM 节点的`style`属性，以下是他的类型定义：

```ts
interface CSSStyleRule {
  /** Always reads as a `CSSStyleDeclaration` */
  get style(): CSSStyleDeclaration;
  /** Can only write a `string` here. */
  set style(newValue: string);
}
element.style; // 返回值为 CSSStyleDeclaration 对象
element.style = "font-size: 20px; color: green;"; // 可以赋值为字符串
```

以上代码中，我们可以看到`CSSStyleDeclaration`是一个对象类型，而`string`是一个字符串类型，它们之间是不兼容的。

因此，在`5.1`版本之后，TypeScript 放宽了这个限制，允许`set`和`get`的类型不兼容，但是需要在定义的时候 **显式指定** 对应类型。

#### 3、示例

例如以下代码中，我们定义了一个`prop`属性的`getter`和`setter`，方法。

```ts title="demo.ts"
class C {
  _prop = "init value";
  get prop() {
    console.log("读取 prop 属性：", this._prop);
    return this._prop;
  }
  set prop(value) {
    console.log("设置 prop 属性：", value);
    this._prop = value;
  }
}
// 实例化
let c = new C();
c.prop = "123"; // 调用 setter
console.log("打印 prop：", c.prop); // 调用 getter
```

输出：

```console
设置 prop 属性： 123              demo.ts:8
读取 prop 属性： 123              demo.ts:4
打印 prop： 123                  demo.ts:15
```

### 4、属性索引

类允许定义属性索引，可以定义属性和方法的索引签名。

下面示例中，`[prop:string]`表示所有属性名类型为字符串的属性，它们的属性值要么是布尔值，要么是返回布尔值的函数。

```ts
class MyClass {
  [prop: string]: boolean | ((prop: string) => boolean);

  getResultById(value: string) {
    return value === "abc";
  }
}
let a: MyClass = {
  prop1: true,
  prop2: (val: string) => val.length > 0,
  getResultById(val: string) {
    return val === "abc";
  },
};
```

注意：类的方法实际上是一种特殊的属性，即属性值为函数的属性，所以属性索引的类型定义也涵盖了方法。如果一个对象同时定义了属性索引和方法，那么属性索引必须包含方法的类型。

```ts twoslash
// @errors: 2411 2564
class MyClass {
  [prop: string]: boolean;
  getResultById(value: string) {
    return value === "abc";
  }
}
```

### 3、`readonly` 修饰符

### 4、可访问性修饰符

类的内部成员的外部可访问性，由三个可访问性修饰符（access modifiers）控制：`public`、`private`和`protected`。

- `public`（默认）：成员可以被类的内部、子类、实例访问。如果省略不写，则默认为`public`。
- `protected`：成员可以在类的内部和子类中访问。
- `private`：成员只能在类的内部访问。

::: info ES2022 引入了新的私有成员写法 `#propName`，它的作用和`private`基本相同，我们会在后文中介绍。
:::

他们的区别：

| 成员特性 | public   | protected | private  |
| -------- | -------- | --------- | -------- |
| 类的外部 | 无法访问 | 无法访问  | 无法访问 |
| 类的内部 | 可以访问 | 可以访问  | 可以访问 |
| 类的实例 | 可以访问 | 无法访问  | 无法访问 |
| 子类内部 | 可以访问 | 可以访问  | 无法访问 |
| 子类实例 | 可以访问 | 无法访问  | 无法访问 |

#### 1、公开成员 `public`

`public`修饰符表示该成员是公开成员。

类的内部、类的实例对象、子类内部、子类的实例可以 访问这个成员，外部无法访问。

```ts
class A {
  public x = "x";
}
class B extends A {
  public y = "y";
  get_paren_x() {
    return this.x;
  }
}
const b = new B();
b.x; // "x"
b.y; // "y"
b.get_paren_x(); // "x"
B.y; // 报错 类型“typeof A”上不存在属性“x”。
```

上述代码中，父类`A`的属性`x`是公开成员，子类`B`的属性`y`是公开成员。子类`B`可以访问父类`A`的公开成员`x`，类的实例`b`也可以访问父类`A`的公开成员`x`。

#### 2、受保护成员 `protected`

`protected`修饰符表示该成员是受保护成员。

类的内部、子类内部可以访问这个成员，类的实例对象、子类的实例无法访问这个成员，外部无法访问这个成员。

```ts twoslash
// @errors: 2445 2339
class A {
  protected x = "x";
}
class B extends A {
  protected y = "y";
  get_paren_x() {
    return this.x;
  }
}
const b = new B();
b.get_paren_x(); // "x"
b.y; // 报错 类型“B”上不存在属性“y”。B.x; // 报错 类型“typeof A”上不存在属性“x”。
```

#### 3、私有成员 `private`

`private`修饰符表示该成员是私有成员。

类的内部可以访问这个成员，类的实例对象、子类的实例无法访问这个成员，外部无法访问这个成员。

```ts twoslash
// @errors: 2341 2339
class A {
  private x = "x";
}
class B extends A {
  private y = "y";
  get_paren_x() {
    return this.x; // 报错 属性“x”为私有属性，只能在类“A”中访问。
  }
}
const b = new B();

b.y; // 报错 类型“B”上不存在属性“y”。
```

#### 4、私有成员写法 `#propName`

严格地说，`private`定义的私有成员，并不是真正意义的私有成员。一方面，编译成 JavaScript 后，`private`关键字就被剥离了，这时外部访问该成员就不会报错。另一方面，由于前一个原因，TypeScript 对于访问`private`成员没有严格禁止，使用方括号写法（`[]`）或者`in`运算符，实例对象就能访问该成员。

```ts
class A {
  private x = 1;
}

const a = new A();

if ("x" in a) {
  console.log(a["x"]); // 1
}
```

由于`private`存在这些问题，加上它是 ES2022 标准发布前出台的，而 ES2022 引入了自己的私有成员写法`#propName`。因此建议不使用`private`，改用 ES2022 的写法，获得真正意义的私有成员。

例如下述代码中，采用了 ES2022 的私有成员写法（属性名前加`#`），TypeScript 就正确识别了实例对象没有属性`x`，从而报错。

```ts twoslash
// @errors: 7053
class A {
  #x = 1;
}

const a = new A();
a["x"]; // 报错
```

#### 5、私有构造函数

构造方法也可以是私有的，这就可以避免使用`new`命令生成实例对象，只能在类的内部创建实例对象。

一般用在==单例模式==中，这时一般会有一个静态方法，充当工厂函数，强制所有实例都通过该方法生成。

下面代码中：通过私有构造方法，实现了单例模式。想要获得 `Singleton` 的实例，不能使用`new`命令，只能使用`getInstance()`方法。每次使用都会返回同一个实例，这就确保了 Singleton 类只有一个实例。

```ts
class Singleton {
  private static instance?: Singleton;

  private constructor() {}

  static getInstance() {
    if (!Singleton.instance) {
      Singleton.instance = new Singleton();
    }
    return Singleton.instance;
  }
}

const s = Singleton.getInstance();
```

### 7、静态成员

#### 1、`static`关键字

类的内部可以使用`static`关键字，定义静态成员。

静态成员是只能通过类本身使用的成员，不能通过实例对象使用，类似于：`Array.from()`、`Array.isArray()`、`Math.PI`等内置方法。

静态成员可以是属性、也可以是方法、甚至是属性索引。

```ts twoslash
// @errors: 2576
class MyClass {
  static x = 0;
  static printX() {
    console.log(MyClass.x);
  }
}

MyClass.x; // 0
MyClass.printX(); // 0
// 实例化对象
let myClass = new MyClass();
myClass.x;
myClass.printX();
```

#### 2、结合可访问性修饰符

`static`关键字前面可以使用 `public`、`private`、`protected` 修饰符。

下述代码中，静态属性`x`前面有`private`修饰符，表示只能在`MyClass`内部使用，如果在外部调用这个属性就会报错。

```ts
class MyClass {
  private static x = 0;
}

MyClass.x; // 报错
```

`public`和`protected`的静态成员可以被继承。

```ts
class A {
  public static x = 1;
  protected static y = 1;
}

class B extends A {
  static getY() {
    return B.y;
  }
}

B.x; // 1
B.getY(); // 1
```

上面示例中，类`A`的静态属性`x`和`y`都被`B`继承，公开成员`x`可以在`B`的外部获取，保护成员`y`只能在`B`的内部获取。

#### 3、静态私有属性 `stastic #propName`

静态私有属性也可以用 ES6 语法的`#`前缀表示，上面示例可以改写如下。

```ts
class MyClass {
  static #x = 0;
}
```

**二者区别：**

| **特性**   | **显式类**                                 | **匿名类**                 |
| ---------- | ------------------------------------------ | -------------------------- |
| 类名       | 有正式类名（如 Person）                    | 无正式类名，只有变量名     |
| 类型系统   | 类名即类型（Person 类型）                  | 结构类型（通过变量名引用） |
| 可访问性   | 可在定义前引用（hoisting）                 | 必须先定义后使用           |
| 调试信息   | 显示实际类名                               | 显示为 class 或变量名      |
| 继承       | 可直接继承（class Student extends Person） | 只能通过变量名继承         |
| 导出/导入  | 可直接导出类                               | 只能导出变量               |
| 构造函数名 | 与类名相同                                 | 显示为 "class"             |
