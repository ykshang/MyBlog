## 一、简介

### 1、概念

TypeScript 的 ==结构化类型==（Structural Typing） 是其类型系统的核心原则之一，也称为 “鸭子类型”（Duck Typing）。它强调类型的兼容性由实际的结构（属性和方法）决定，而非显式的类型声明名称。

结构化原则主要用在类型检查上，类型检查是理解 TypeScript 类型系统的关键概念。

### 2、主要场景

类型检查主要发生在两类场景：

- ==值传递==：函数的参数传递、返回值类型检查；变量的声明、赋值；类的实例化，。
- ==类型操作==：属性索引；类的继承；接口的实现、合并、继承；类型别名的合并；类型转化、类型断言；

上述场景都需要涉及到类型检查，或者说类型兼容性检查。

## 二、详细介绍

举个例子，有这么一段代码：

```console
A = B
```

上述代码，我们可以理解为：类型上 `A` 兼容 `B`，`B` 的值符合 `A` 的要求。

接下来我们使用 `A` 和 `B` 分别表示两个类型，来详细的介绍一下类型检查。

### 1、字面量检查

字面量检查主要检查 `B` 的成员列表，是否符合了 `A` 的要求：

- `A` 里列出的成员，`B` 里必须有。
- `B` 包含的成员不能缺少，但是可以多出来。

例如下边的代码，`B` 中的`sex` 是 `A` 中没有的。但是 `A` 里的成员 `name` 和 `age`，`B` 中都有，所以 `A` 兼容 `B` 的。

```ts
type A = {
  name: string;
  age: number;
};

type B = {
  name: string;
  age: number;
  sex: string;
};
```

### 2、属性的类型检查

属性的类型检查主要检查同名属性的类型是否兼容，一般来说类型更加严格的属性，越容易被兼容，`any` 例外。

例如下边的代码，`B` 中的 `name` 都比 `A` 中的 `name` 更加严格。

```ts twoslash
// @errors: 2416
// @noErrors: 2564
class A {
  name: string | number;
  age: number;
}

class B extends A {
  declare name: string;
  declare age: string;
}
```

### 3、方法的类型检查

- 方法参数列表：参数可以越来越少，甚至可以不传，类型要求更加严格。但是在调用的时候参数必须和定义完全一致。
- 返回值类型：返回值的类型要求更加严格。


```ts
interface A {
  get(name: string, n: number, aa: string): boolean | number;
}

class B implements A {
  get(name: string, n: number): boolean {
    // s 的类型是 any
    return true;
  }
}
class C extends B {
  get(name: string): boolean {
    return true;
  }
}
class D extends C {
  get(): boolean {
    return true;
  }
}
let c = new C();
let d = new D();
c.get("1");
d.get();
```

### 4、可选性检查

可选的可以变必选，但是必选的不能变可选。

```ts twoslash
// @errors: 2430
interface Options {
  name: string;
  debug?: boolean;
  log: string;
}

// 子接口可以使可选属性变为必需
interface StrictOptions extends Options {
  name: string;
  debug: boolean; // 现在是必需的
  log?: string;
}
```

### 5、可访问性检查

可访问性主要涉及到类的继承。子类中，成员的可访问性也可以和父类保持一致，也可以更宽松。但是子类不能定义父类的同名私有成员。

主要表现在以下几个方面：

- 父类中是 `public`，在子类中只能为 `public`，不可以被改变。
- 父类中是 `protected`，在子类中可以为 `public` 和 `protected`，可以被提升。
- 子类不可以定义父类中已存在的同名私有 `private` 成员。

```ts twoslash
// @errors: 2415
class Parent {
  public a: string = "a";
  protected b: string = "b";
  private c: string = "c";
}
class Child1 extends Parent {
  protected declare a: string;
}
class Child2 extends Parent {
  public declare b: string;
  // @annotate: b 的类型变宽松
}
class Child3 extends Parent {
  private declare c: string;
}
```

### 6、静态属性检查

静态属性的检查和属性类型检查类似。但是静态属性和普通属性的检查是分开的。
静态属性只和同名的静态属性去匹配检查。

```ts
class Options {
  version: number;
  static version: string = "1.0.0";
}

// 子接口可以使可选属性变为必需
class StrictOptions extends Options {
  declare version: number;
  static version: string = "1.0.1";
}
```

### 7、只读属性不检查

只读属性不影响类型检查。

```ts
class Options {
  readonly version: number;
  name: string;
}
Class StrictOptions extends Options {
  declare version: number;
  declare readonly name: string;
}
```
