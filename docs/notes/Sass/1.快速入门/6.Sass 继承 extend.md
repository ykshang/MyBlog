在 Sass 中，**继承（`@extend`）** 是一种复用样式的强大工具，可以减少代码冗余并提升可维护性。以下是关键知识点和示例：

## 一、基本用法：`@extend`

通过 `@extend` 指令，一个选择器可以继承另一个选择器的样式。

```scss
.button-base {
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
}

.primary-button {
  @extend .button-base; // 继承基类
  background-color: blue;
  color: white;
}

// 编译后的 CSS：
.button-base,
.primary-button {
  padding: 10px 20px;
  border: none;
  border-radius: 4px;
}
.primary-button {
  background-color: blue;
  color: white;
}
```

## 二、占位符选择器：`%`

占位符（以 `%` 开头）不会直接编译到 CSS 中，只有被继承时才会生成。

```scss
%message-shared {
  border: 1px solid #ccc;
  padding: 10px;
}

.success-message {
  @extend %message-shared;
  border-color: green;
}

// 编译后的 CSS：
.success-message {
  border: 1px solid #ccc;
  padding: 10px;
}
.success-message {
  border-color: green;
}
```

## 三、继承 vs 混合器（Mixins）

- **`@extend`**  
  适合共享**完全相同**的样式，生成紧凑的群组选择器。
- **`@mixin`**  
  适合需要**动态传参**或生成不同样式的情况。

```scss
// 混合器示例：支持动态值
@mixin button-theme($color) {
  background-color: $color;
  &:hover {
    background-color: darken($color, 10%);
  }
}

.danger-button {
  @include button-theme(red);
}
```

## 四、注意事项

1. **避免过度继承**  
   长继承链可能导致 CSS 冗余，尤其在深层嵌套时。
2. **作用域限制**  
   确保被继承的选择器在作用域内可用（如不在媒体查询内部）。

3. **与混合器结合使用**  
   根据场景灵活选择：静态复用用 `@extend`，动态逻辑用 `@mixin`。

## 五、最佳实践

- **使用占位符**：当基类不需要单独使用时，优先用 `%` 避免冗余代码。
- **保持简单**：继承链不宜过长，确保代码可读性。
- **性能考量**：在大型项目中，过度继承可能影响编译速度，需适度使用。

通过合理运用 `@extend` 和占位符，你可以高效管理样式复用，写出更简洁、灵活的 Sass 代码。
